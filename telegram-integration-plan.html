      "id": 123456789,  ← session_id
      "type": "private"
    },

  }

Step-by-Step Implementation

Phase 1 — Create the Telegram Bot

  • 1
    Open Telegram and find @BotFather

    Search for @BotFather in Telegram → open chat → click Start

  • 2
    Create new bot

    Send /newbot → enter a name (e.g. DockerAgent) → enter a username ending in "bot" (e.g. docker_ops_bot)

  • 3
    Save the Bot Token

    BotFather replies with a token like: 7412638901:AAFxxxxxxxxxxxxxxxxxxxxxxxx. Save this — you'll need it in n8n.

Phase 2 — Add Telegram Credential in n8n

  • 1
    Open n8n credentials

    Go to http://localhost:5678 → top right → Credentials → Add Credential → search Telegram

  • 2
    Paste the Bot Token

    Paste the token from BotFather → Save. Name it "DockerAgent Bot"

Phase 3 — Update n8n Workflow

  • 1
    Delete the Webhook trigger node

    Open the workflow editor → click Webhook node → Delete

  • 2
    Add Telegram Trigger node

    Add node → search Telegram Trigger → select credential → set Updates to message → connect to Edit Fields

  • 3
    Update Edit Fields node

    Change session_id to: {{ $json.message.chat.id }}
    Change message to: {{ $json.message.text }}

  • 4
    Update Code node

    Add chat_id to the Code node output so the Telegram reply node can use it

  • 5
    Delete the last HTTP Request node (Evolution API)

    Click the last HTTP Request node (POST evolution-api) → Delete

  • 6
    Add Telegram Send Message node

    Add node → search Telegram → select Send Message action → credential: DockerAgent Bot → Chat ID: {{ $json.chat_id }} → Text: {{ $json.text }}

  • 7
    Save and Publish

    Connect all nodes: Telegram Trigger → Edit Fields → docker-agent → Code → Telegram → Save → Publish

Phase 4 — Test

  • 1
    Find your bot on Telegram

    Search for your bot username (e.g. @docker_ops_bot) → open chat → click Start

  • 2
    Send initial message

    Send . → expect: "Hello! I'm DockerAgent. What do you want to do?"

  • 3
    Test full add flow

    Reply add → follow the prompts (name → image → port → ...) → verify service appears in docker ps

Updated Code Node (JavaScript)

Only the Code node changes slightly — add chat_id to the output so the Telegram node can use it. 
const data = $input.first().json;
const text = (data.text || data.next_question || '').trim();
const chat_id = data.session_id || '';

return [{
  json: {
    chat_id: chat_id,
    text: text
  }
}];
